存储落点（数据字典对应）
表来源写入策略
road_link / road_link_coord需求1 全量路网 areaLinkPub按 link_id upsert；坐标整段删后重插
traffic_status需求1 全量实时路况按 link_id upsert，只存最新并刷新时间；分段聚合到顶层
transit_line_raw需求2 阶段一追加存档原始响应（含 logs/transit_raw 文件）
transit_segment需求2 阶段二（经 SDK）按 (line_name, direction) 整段替换，含 reverse_coords 与 line_track
要点：① 需求1 由 cron 长驻 daemon 驱动；路网响应大，可配 stream 模式边下边解析省内存。② 公交三接口走不同主机，token 需 MD5 签名、后续请求带 Bearer。③ 阶段二的 SDK 以已落地的路网（road_link_coord）为底图做空间匹配，所以须先跑需求1 灌满路网，再跑 transit-build。
