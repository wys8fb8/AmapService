各阶段判据与阈值（与代码一致，均为可配参数）

阶段做什么判据 / 阈值（配置项）解决的真实问题
道路等级惩罚代价里给高等级路加罚分，距离接近时让地面路胜出高速rc0 +60m、高架rc6 +25m（roadclass_penalty）47路误匹配「内环高架路」，地面「中山北路」更近反被压住
方向感知候选里优先与 GPS 同向的 link夹角 ≤ reverse_angle_deg(90°) 视为同向分车道误匹配对面车道、被判逆行
迟滞保持仅当当前 link 仍接近最优才保持，否则切更优路差值 < hysteresis_m(15m)误上高架后一路粘着高架不下来
连通优先优先选与上一段共端点的 link端点距 < connect_gap_m(8m)路口处碎跳到平行/横街，序列不连贯
方向翻转拆段同 link 上行进方向翻转则拆为两段opposed 翻转即拆不分车道折返被合并、reverse 投票互相抵消
去绕路孤岛删「插入它反而绕远」的离线段多绕 > detour_gap_m(80m)617m 长穿越路被单点误匹配，两头甩出 ~270m
去偏离突兀删整体偏离原轨迹的段顶点偏离 > track_fit_m(25m)个别段离原轨迹很远、突兀
去对面车道/反向删与路线方向相反的段（带折返闸门）相差 > against_track_deg(120°)，且该处轨迹未折返对面车道反向段；同时保护单环线/掉头的合法返程
去折返闭环删「绕出去又绕回起点」的整段环起点回到更早终点 < loop_return_m(10m)，窗口 8 段二次匹配在缺口处兜圈匹配对面车道（路网不通）
去垂直 jut删 GPS 漂移到垂直横街的段（带偏离闸门）偏转 > jut_deg(60°)+前后同向 < jut_neighbor_deg(45°)+偏离 > jut_offtrack_m(15m)「安国路」垂直横街漂移段（删）；「嘉陵路」在轨迹上的真实绕行（保留）
二次匹配首遍结果连成干净轨迹再匹一遍，补 GPS 漂移缺口refine_passes(2)、densify_step_m(15m)路段间断一截子（连接 link 上无 GPS 点）
设计原则：① 不裁剪——输出完整路网路段（真实路网粒度够细，路段即段）；② 一切「删/留」判据都以原始 GPS 轨迹为准绳；③ 清理只删「确实离开轨迹/反向/绕路」的段，对「真转弯、单环线双向、在轨迹上的绕行」一律保留；④ 所有阈值均为 TrackConverter / config.sdk 可配参数，便于按城市路网微调。
双向经过：不分车道（单 link 双向）的路来回各经一次 → 同 link 出现两次（去 reverse=False、回 reverse=True）；分车道则去回本就是两条独立 link。
