# 站间路况分段 SDK 设计

> 时间：2026-06-08 · 归属：需求3（GPS↔路段 SDK）扩展
> 前置：需求2 已实现「线路轨迹 → 站有序的路段链」并落库 `transit_segment`。

## 1. 目标

给前端渲染「相邻两站之间这一段线」的路况配色用。这一段由若干路段（link）拼成，每个 link 路况不同，前端按每个 link 的**实际长度占比**画出不同颜色、不同长度的色块。

提供两个 SDK 方法：

- **方法一**：给定线路对象 + 方向 + 站级号，返回「上一站 → 该站」区间内的路段列表，每条带路况与长度占比。
- **方法二**：方法一的全量批量，输入整个线路对象，返回所有方向、所有站间区间。

## 2. 接口

新增 [amap_service/sdk/station_traffic.py](../../../amap_service/sdk/station_traffic.py)，类 `StationTrafficResolver`（构造接 `engine`，与 `TrackConverter` 同风格）。在 [amap_service/sdk/__init__.py](../../../amap_service/sdk/__init__.py) 导出。

```python
class StationTrafficResolver:
    def __init__(self, engine, sample_step_m: float = 4.0, default_state: int = 1): ...

    def station_section(self, line_object: dict, direction: int, level_id: int) -> list[dict]:
        """方法一：单个站间区间。
        -> [{"link_id": int, "state": int, "pct": int}, ...]
        区间内所有 pct 之和恒为 100（最大余额法取整）。
        """

    def line_sections(self, line_object: dict) -> dict:
        """方法二：整条线路全部站间区间（方法一的批量）。
        -> { direction: [ {level_id: [ {link_id, state, pct}, ... ]}, ... ] }
        例：{ 0: [ {2: [...]}, {3: [...]}, ... ], 1: [ {2: [...]}, ... ] }
        缺失的方向不出现（单环线只有方向 0）。
        """
```

约定：

- `line_object` 即 `GetRoadLineEntity` 的响应，与 [parse_line_tracks](../../../amap_service/parsing/transit.py) 吃的是同一结构（带 `Data`）。方法内从中取 `LineName` 与 `UpObject/DownObject.Stations`。
- `direction`：`0=上行/单环`，`1=下行`；对象映射 `0→UpObject`、`1→DownObject`。
- `level_id`：站级号，从 **2** 起；区间 = 第 `level_id-1` 站 → 第 `level_id` 站。

## 3. 数据来源

| 数据 | 来源 | 说明 |
|---|---|---|
| 有序路段链 | `transit_segment`（按 `line_name`+`direction`，`seq` 升序） | 已清洗/补洞/裁剪，含每段 `line_track`（已按行进方向，`reverse_coords` 已应用） |
| 站点坐标 | `line_object` 的 `Up/DownObject.Stations` | 每站 `LevelId`、`Lon02`、`Lat02` |
| 路况 | `traffic_status`（`link_id` → `state`） | 一次 `IN` 批量查链上全部 link；查不到默认 `state=1`（畅通） |

`state` 枚举（见 `demo/数据字典.md`）：`1=畅通, 2=缓行, 3=拥堵, 4=严重拥堵, 5=未知`。默认值取 `1`。

两个方法都**读已落库的 `transit_segment`**（不在调用时重新转换 `LineLonLat`），保证与落库结果一致且快。

## 4. 核心算法

### 4.1 拼链 + 弧长/归属标注

把该方向的 `transit_segment` 按 `seq` 拼成一条折线。**不去重接缝点**，对每个顶点记 `(累计弧长 arc, 所属 link_id)`。每条 link 因此对应一个弧长区间 `[arc_start, arc_end]`（接缝处弧长零步进，区间边界干净）。链总长 `T` = 末顶点 arc。

> 关键：同一 `link_id` 在回头路场景会出现两遍 —— 即两个**独立**的弧长区间。整套算法靠弧长区间定位，不靠 `link_id` 标识，故不会混。

### 4.2 站点单调对齐（消歧核心）

站点按 `LevelId` 有序，但坐标在空间上可能近乎重合（环线/掉头单车道/上下行对开/路对面）。**不做全局最近点投影**，改为**弧长非递减约束的 DP 对齐**：

1. 沿链按 `sample_step_m`（默认 4m）采样得 `[(arc, point), ...]`。
2. `dp[i][j]` = 把第 `i` 站落到采样点 `j`、且 `j.arc ≥ 第 i-1 站所选 arc` 的前提下，各站到其落点距离之和的最小值。
3. 回溯得到每站弧长位置 `s_1 ≤ s_2 ≤ … ≤ s_N`。

效果：

- 第 2~8 站占满去程及之后的弧长 → **第 9 站只能落到回程那一遍**，哪怕去程在空间上更近、甚至是同一条 `link_id`。
- 第 1 站不会被吸到环线末端：那样会把后续站逼入死角、总距离爆炸，DP 自然把它锚在前段。
- 复杂度 `O(站数 × 采样数)`，几十站 × 上千采样，毫秒级。

> 这正是「靠站沿链单调前进消歧，而非靠空间最近」，对应并解决回头路/对开/环线难点（与此前 `_drop_against_track` 踩的去/回程重叠是同一类坑）。

### 4.3 切区间 + 算占比

对 `k = 2..N`，区间 `[s_{k-1}, s_k]`：

1. 对链上每条 link 区间 `[a, b]`，重叠长度 `len = max(0, min(s_k, b) − max(s_{k-1}, a))`；`len > 0` 的按链顺序收集。
2. `state = traffic.get(link_id, default_state)`。
3. `pct`：对各 `len` 用**最大余额法**取整到和=100 —— 先 `floor(len/总长*100)`，再把剩余的 `100 − Σfloor` 个名额按小数部分从大到小各补 1。

输出按链顺序。同一 `link_id` 在不同区间各列一条；若某区间恰好跨同一 link 的两遍（区间内出现回头），也各列一条（不去重）。

## 5. 边界处理

- 该方向 `transit_segment` 为空（未跑 build / 单环无下行）→ 方法一返回 `[]`；方法二跳过该方向。
- `level_id < 2` 或 > 站数 → 方法一返回 `[]`。
- 站投影点超出链首/尾 → arc 钳到 `[0, T]`。
- 零长区间（`s_{k-1} == s_k`，两站投到同一点）→ 取该点所在 link，输出单条 `[{link_id, state, 100}]`（保证前端有内容可画且和=100）。
- 链上 link 在 `traffic_status` 无记录 → `state = default_state`（1）。

## 6. 测试

新增 `tests/sdk/test_station_traffic.py`（合成几何为主，真实数据抽验）：

1. 直链、站落在已知位置 → pct 精确。
2. 站落 link 中间 → 该 link 在相邻两区间各出现一条，占比按两侧长度分。
3. **回头单车道**：同一 link 在链中出现两遍，两站贴同一路但分属去/回程 → 单调对齐各落对腿（核心回归用例）。
4. **对开分离**：去/回程是两条对向 link → 各落对腿。
5. 路况缺失 → 默认 1。
6. 最大余额法和=100（如三等分 → 33/33/34）。
7. 零长区间兜底。

真实数据用 192/47/89（`logs/transit_raw/` 有线路对象 JSON）跑方法二做 sanity：每区间 pct 和=100、link 顺序与 `transit_segment` 一致。

## 7. 不做（YAGNI）

- 不新建站点表：站坐标都从入参 `line_object` 取。
- 不在调用时重新匹配/转换轨迹：只消费已落库的 `transit_segment`。
- 不返回每个 link 的几何坐标：前端按 `state` 取色、按 `pct` 取长度画比例条即可，无需真实折线。
- 不合并相邻同 `state` 的不同 link：输出按 link 粒度逐条列出。
