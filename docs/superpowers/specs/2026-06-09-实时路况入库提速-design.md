# 实时路况入库提速与实时服务设计

> 时间：2026-06-09 · 归属：需求1（地图数据落地）+ 对第三方实时供数
> 现状问题：全量实时路况约 40 万行，每 2 分钟抓一次，但入库要好几分钟——超过抓取周期，数据对第三方不再实时。

## 1. 根因

[upsert_traffic_status](../../../amap_service/db/repositories.py) 是**逐行 upsert**：

```python
for row in batch:
    stmt = _upsert_stmt(...).values(**values)   # 每行重新编译
    conn.execute(stmt)                            # 每行一次往返
```

40 万行 = 40 万次语句编译 + 40 万次 execute 往返，外加每批一个仅为统计 inserted/updated 的 `existing` 查询、每行一次 `func.current_timestamp()`。这是逐行反模式，与数据量无关。

同样地，[pipelines/traffic.py](../../../amap_service/pipelines/traffic.py) 的 Redis 路径也是逐行：incremental 每个 link 一次 `get`+`set`，snapshot 每个 link 一次 `set`——40 万次 Redis 往返。只改 DB 不改 Redis 会把瓶颈转移到缓存。

## 2. 目标与约束

- 入库从「分钟级」降到「秒级」，稳定落在 2 分钟周期内。
- 第三方**经我们的代码/接口**按 link 单点/批量查实时路况，要求**亚毫秒、永远最新**。
- 同一套代码**同时兼容 SQLite（开发）与 MySQL（生产）**。
- Redis 已启用，可依赖。
- `traffic_status` 是**全量、latest-only**（按 link_id 覆盖），无需历史。

## 3. 设计（五部分）

### A. DB 写入提速（核心）

重写 `upsert_traffic_status`：

- **双方言 executemany**：一批一条 `INSERT ... ON CONFLICT DO UPDATE`（SQLite）/ `INSERT ... ON DUPLICATE KEY UPDATE`（MySQL），用 `conn.execute(stmt, params_list)` 批量执行；批大小默认 5000（可配）。
- `updated_at`：整批用**一个** Python 端时间戳值（放进每行参数），不再每行 `func.current_timestamp()`。
- 去掉每批为统计而做的 `existing` 查询；返回 `{written, failed}`（latest-only 不需要 inserted/updated 拆分）。
- `set_` 用 `excluded`/`inserted` 引用新值，executemany 下成立。

`db/engine.py`：SQLite 连接事件挂 PRAGMA（仅 sqlite 方言生效）：

```
journal_mode=WAL, synchronous=NORMAL, temp_store=MEMORY, cache_size=-65536  (约 64MB)
```

> 预期：40 万行从分钟级降到秒级；MySQL 走 executemany ON DUPLICATE，SQLite 走 WAL + executemany ON CONFLICT。

### B. Redis 路径批量化

`cache/client.py` 给 `RedisCache` 增加批量接口（底层 `pipeline`）：

- `mget(keys: list[str]) -> list[Optional[str]]`
- `mset(mapping: dict[str, str], ttl: Optional[int] = None)`

`NoOpCache` 给对应空实现（`mget` 返回全 None 列表、`mset` 无操作），保持「禁用时无 if 分支」契约。

`pipelines/traffic.py`：
- incremental：用一次 `mget` 取所有 `traffic:sig:{link_id}` 签名，内存比对挑出变化行，再 `mset` 批量回写签名。
- snapshot：变化行用 `mset` 批量写 `traffic:latest:{link_id}`。

### C. 减量（incremental 默认开启）

打开 `redis.uses.incremental_detection`：只把 `(speed,state,travel_time)` 变化的 link 写 DB + 刷新签名/快照。两次抓取间多数 link 不变 → DB 与 Redis 写入量大幅下降。与 A 叠加后稳定在秒级。

> 说明：incremental 下未变化 link 的 `updated_at` 不刷新、Redis 快照不重写——值未变，无需刷新；读取侧（D）对 Redis 缺失回落 DB，保证一致。

### D. 实时热读 SDK（第三方查询入口）

新增 `amap_service/sdk/traffic_query.py`：

```python
class TrafficReader:
    def __init__(self, engine, cache=None): ...
    def get_latest_traffic(self, link_ids: list[int]) -> dict:
        # -> { link_id: {"state", "speed", "travel_time", "traffic_time"} }
        # 先 Redis mget(traffic:latest:{id})，命中即用；未命中的一次 IN 查 DB 回落。
        # Redis 与 DB 都查不到的 link 不出现在结果里（缺省由调用方决定，与 StationTrafficResolver 的 default_state 一致）。
```

- Redis 命中走 `json.loads`；未命中集合做一次 `traffic_status` 的 `IN` 查询回落。
- 在 `sdk/__init__.py` 导出 `TrafficReader`。
- 第三方按 link 单点/批量查走它：Redis 命中亚毫秒、且是最后一次抓取值。

### E. 双引擎兼容

- `_upsert_stmt`/批量写入按 `conn.dialect.name` 分支（已有 sqlite/mysql 两支），扩成 executemany 形式。
- PRAGMA 只在 sqlite 连接生效；MySQL 连接不挂。
- 测试对 SQLite 跑全链路；MySQL 分支至少做语句构造/方言选择的单测（不依赖真实 MySQL 服务）。

## 4. 不在本次范围（YAGNI）

- 不引入 staging 表 + 原子切换：第三方按 link 单点查，逐行覆盖已天然一致（读到的某 link 要么旧要么新，不会半行）；全量一致快照非当前需求。
- 不改 `StationTrafficResolver._load_traffic`：它服务内部 section 叠加，已是一次 `IN` 查询，无瓶颈；保持读 DB。
- 不做历史留档表：traffic_status 仍 latest-only。
- 不做对外 HTTP API：本次只提供 `TrafficReader` SDK；接口层（若需要）是后续独立工作。

## 5. 测试

- `tests/db/test_repositories.py`：`upsert_traffic_status` 批量写入正确性（插入 + 覆盖 + 64 位 link_id + 返回 {written, failed}）；大批量（如 5k+ 行跨多批）一次写入正确。
- `tests/db/test_engine.py`：SQLite PRAGMA 生效（连上后 `PRAGMA journal_mode` 返回 wal 等）。
- `tests/cache/test_client.py`：`RedisCache.mget/mset`（用 fakeredis）；`NoOpCache.mget/mset`。
- `tests/pipelines/test_traffic.py`：incremental 用 mget 比对只写变化行；snapshot 用 mset 批写；无缓存时回落整批写入；返回 stats。
- `tests/sdk/test_traffic_query.py`：`get_latest_traffic` Redis 命中、DB 回落、混合命中、空输入；禁用缓存时纯 DB。
- 全量回归保持绿。

## 6. 落地与配置

- `config.yaml` 的 `redis.enabled: true` 且 `redis.uses.incremental_detection: true` / `latest_traffic_snapshot: true`。
- 批大小：不新增配置项，`upsert_traffic_status(..., batch_size: int = 5000)` 用函数默认值即可（保持调用面不变）。
- `docs/命令说明.md` 增补：实时路况的 Redis 加速开关与第三方查询走 `TrafficReader` 的说明。
